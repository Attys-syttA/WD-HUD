# Codex Task Files

Task and plan lifecycle:

- active plans: `docs/codex-tasks/plans/pending/active/`
- not-started plans: `docs/codex-tasks/plans/pending/not-started/`
- reference material: `docs/codex-tasks/plans/pending/reference/`
- completed plans: `docs/codex-tasks/plans/done/`

Active plans must start with a short status block:

- Status
- Current repo relation
- Completed parts
- Open parts
